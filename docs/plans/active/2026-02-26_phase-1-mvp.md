## Контекст

Необходим рабочий Telegram-бот планировщик задач с AI-парсингом естественного языка. Это первая фаза из 6 в плане развития TG Planer (см. [docs/DOMAIN.md](../../DOMAIN.md)). Фокус — минимальный продукт: создание задач текстом, подтверждение через inline-кнопки, проекты, напоминания и деплой на Vercel.

### Вопросы и ответы

*Вопросы, заданные при создании плана:*

**Q:** Какой ORM использовать — Prisma или Drizzle?
**A:** Drizzle ORM — лёгкий, SQL-like API, отличная TypeScript типизация, быстрее Prisma на serverless.

**Q:** Какую библиотеку для Telegram Bot API?
**A:** grammy — современная, TypeScript-first, нативная поддержка webhooks.

**Q:** Polling или webhook?
**A:** Webhook — Vercel serverless не поддерживает long-polling.

## Решение

Монолитное Next.js приложение с App Router: API route для Telegram webhook, Drizzle ORM для PostgreSQL, абстракция LLM-провайдера с первой реализацией на OpenAI GPT-4o-mini для парсинга задач. Деплой на Vercel с Cron Jobs для напоминаний.

### Альтернативы

1. **Отдельный бот-сервер (Express/Fastify)** — усложняет деплой, два сервиса вместо одного
2. **Prisma ORM** — тяжелее, медленнее на serverless из-за query engine
3. **Polling вместо webhook** — не работает на Vercel serverless

## Последствия

### Позитивные
- Рабочий MVP с AI-парсингом за минимальное время
- Единый фреймворк (Next.js) для бота и будущего Mini App
- LLM-абстракция позволяет менять провайдера в будущем
- Drizzle ORM — лёгкий, SQL-like, быстрый на serverless

### Риски
- Vercel Cron Jobs: минимальный интервал 1 минута — напоминания с точностью ±1 мин
- Cold start на Vercel может увеличить время ответа бота
- Зависимость от OpenAI API — нужен fallback

### Технический долг
- In-memory хранение распарсенных задач не переживает рестарт (допустимо для MVP)
- Нет тестов (добавить в следующих итерациях)
- Нет rate limiting на webhook

## Тестирование
