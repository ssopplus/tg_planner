# TG Planer — Domain Document

> Результат дискавери-сессии. Описание продукта, ключевые решения, архитектурные ограничения и план развития.

---

## 1. Обзор продукта

**Telegram-бот планировщик задач** с Mini App интерфейсом для управления повседневными и рабочими задачами, включая умный список покупок.

### Целевая аудитория
- Личное использование (single user)
- Multi-user заложен в архитектуру на будущее

### Ключевые проблемы, которые решаем
- **Слишком много кликов** для создания/изменения задач (боль Todoist)
- **Нет фокуса на "что делать сейчас"** — непонятно, за что браться в данный момент

### Ключевые принципы
- Минимум действий для создания задачи
- Бот сам формирует фокус дня
- AI помогает, а не мешает

## 2. Способы ввода задач

### 2.1 Быстрый текст с AI-парсингом
Пользователь пишет естественным языком, бот разбирает:
- Название задачи
- Дату/время (жёсткий или мягкий дедлайн)
- Категорию/проект
- Приоритет (если указан)

**Пример:** "купить молоко завтра 18:00" → Задача: купить молоко, Срок: завтра 18:00, Проект: Покупки

### 2.2 Голосовые сообщения
- Распознавание речи (Whisper API ~$0.006/мин)
- Умный парсинг: из одного голосового может создаться несколько задач
- **Пример:** "Завтра надо купить молоко и позвонить маме в три часа" → 2 задачи с датами

### 2.3 Кнопки/меню
- Inline-кнопки в Telegram для пошагового создания
- Для пользователей, которые предпочитают структурированный ввод

### 2.4 Подтверждение создания
Всегда показываем результат парсинга: "Задача: X, Срок: Y. [Верно] [Изменить]"

---

## 3. Структура задач

### 3.1 Проекты (папки)
- Задачи группируются в проекты (Работа, Дом, Покупки и т.д.)
- Канбан-доска внутри проекта — возможна в будущем (Phase 3)

### 3.2 Подзадачи
- Один уровень вложенности с прогрессом ("выполнено 3 из 7")
- Многоуровневая вложенность — возможна в будущем

### 3.3 Приоритеты
- **Ручные уровни:** высокий / средний / низкий (опционально при создании)
- **Автоматическое ранжирование:** бот определяет приоритет по дедлайну, тегам и контексту для формирования фокуса дня

### 3.4 Дедлайны
- **Жёсткие:** конкретная дата + время (до 5 марта 18:00)
- **Мягкие:** "на этой неделе", "когда-нибудь", "в ближайшие 3 дня"

### 3.5 Просроченные задачи
- Автоматический перенос на сегодня
- Утром бот спрашивает: "Задача X просрочена. [Перенести] [Удалить] [Выполнить]"

---

## 4. Список покупок

Отдельный тип проекта с расширенным поведением:

- **Быстрый ввод списком:** "молоко, хлеб, яйца" → 3 позиции
- **Категории товаров:** автогруппировка по отделам (молочное, овощи, бытовая химия)
- **Запоминание частых покупок:** бот предлагает то, что покупаешь регулярно

---

## 5. Планирование дня ("Мой день")

### Утренний дайджест
- Бот присылает топ задач на сегодня (по дедлайнам + приоритетам)
- Включает просроченные задачи с вопросом что с ними делать

### Автоматический "Мой день"
- Бот формирует список на основе дедлайнов и автоприоритетов

### Ручная корректировка
- Пользователь может добавить/убрать задачи из плана на день

### Вечерний итог
- Сводка: сколько сделано, что перенесено на завтра

## 6. Напоминания

| Тип | Описание |
|-----|----------|
| По времени | "Напомни в 18:00" |
| Утренний/вечерний дайджест | План на день / итоги дня |
| Повторяющиеся | "Каждый понедельник в 9:00", "каждый день" |
| Привязка к дедлайну | Напомнить за час/день до срока |

### Уровни уведомлений
- **Тихие** — для некритичных задач
- **Со звуком** — для важных/срочных
- **С inline-кнопками** — [Готово] [Отложить] [Пропустить]

---

## 7. AI-фишки

### 7.1 AI-помощник
- Декомпозиция задач: "разбей на подзадачи" → бот предлагает структуру
- Оптимальный порядок выполнения

### 7.2 Контекстные подсказки
- Бот замечает паттерны поведения
- Пример: "Ты откладываешь эту задачу 3-й раз. Может, разбить на части?"
- Предложения на основе привычек

### 7.3 LLM-провайдер
- Абстракция над LLM — возможность менять провайдера (OpenAI, Anthropic и др.)
- Для парсинга — дешёвая модель (GPT-4o-mini / Haiku)
- Для сложных задач (декомпозиция) — мощнее (GPT-4o / Sonnet)

---

## 8. Интерфейс

### 8.1 Чат-бот (основной ввод)
- Быстрое создание задач текстом/голосом
- Уведомления и напоминания
- Быстрые действия через inline-кнопки

### 8.2 Mini App (управление и обзор)
- Просмотр задач по проектам
- Планирование дня (drag-n-drop)
- Управление проектами и настройками

### 8.3 UI-стек
- **shadcn/ui + Tailwind CSS** — основной фреймворк
- **Telegram UI Kit** — для нативных элементов где уместно

## 9. Технический стек

| Компонент | Технология |
|-----------|------------|
| Язык | TypeScript |
| Фреймворк | Next.js |
| Mini App | Telegram Mini App (WebApp API) |
| БД | PostgreSQL |
| Хостинг | Vercel |
| Распознавание речи | Whisper API (OpenAI) |
| LLM | Абстракция (OpenAI / Anthropic / другие) |
| Авторизация | Telegram WebApp API |

### Данные пользователя
- Telegram ID
- Имя
- Настройки: часовой пояс, время дайджеста, предпочтения

### Язык интерфейса
- Русский (пока единственный)

### Архив
- Выполненные задачи уходят в архив (не удаляются)
- Доступны для просмотра истории

## 11. План развития по фазам

### Phase 1 — MVP (Минимальный продукт)
> Цель: рабочий бот с AI-парсингом и напоминаниями

- [ ] Telegram-бот: создание/удаление/завершение задач через чат
- [ ] AI-парсинг текста: распознавание задачи, даты, времени из естественного языка
- [ ] Подтверждение создания: показ результата парсинга + кнопки [Верно] / [Изменить]
- [ ] Проекты: создание, переключение, просмотр задач по проекту
- [ ] Простые напоминания по времени
- [ ] Дедлайны: жёсткие (дата+время) и мягкие ("на этой неделе")
- [ ] Приоритеты: ручные (высокий/средний/низкий)
- [ ] Авторизация через Telegram WebApp API
- [ ] PostgreSQL: схема данных (users, projects, tasks, reminders)
- [ ] Деплой на Vercel

### Phase 2 — Умный бот
> Цель: голос, умные уведомления, планирование дня

- [ ] Голосовые сообщения: распознавание (Whisper) + умный парсинг нескольких задач из одного сообщения
- [ ] Утренний дайджест: автоматический план на день
- [ ] Вечерний итог: сводка дня
- [ ] "Мой день": автоформирование + ручная корректировка
- [ ] Автоматический приоритет: ранжирование по дедлайну и контексту
- [ ] Просроченные задачи: автоперенос + утренний опрос
- [ ] Повторяющиеся напоминания (каждый день, каждый понедельник и т.д.)
- [ ] Напоминания за N часов/дней до дедлайна
- [ ] Уведомления с inline-кнопками: [Готово] [Отложить] [Пропустить]
- [ ] Тихие/звуковые уведомления по приоритету

### Phase 3 — Mini App
> Цель: полноценный визуальный интерфейс

- [ ] Mini App: список задач по проектам
- [ ] Mini App: просмотр и редактирование задачи
- [ ] Mini App: экран "Мой день" с drag-n-drop
- [ ] Mini App: управление проектами
- [ ] Mini App: настройки (часовой пояс, время дайджеста)
- [ ] Подзадачи: один уровень + прогресс ("3 из 7")
- [ ] Архив выполненных задач

### Phase 4 — Умный список покупок
> Цель: покупки как отдельный smart-проект

- [ ] Тип проекта "Покупки" с расширенным поведением
- [ ] Быстрый ввод списком: "молоко, хлеб, яйца" → 3 позиции
- [ ] Автокатегоризация по отделам (молочное, овощи, химия)
- [ ] Запоминание частых покупок
- [ ] Предложения на основе истории покупок

### Phase 5 — AI-фишки
> Цель: бот становится умным помощником

- [ ] AI-декомпозиция задач: "разбей на подзадачи"
- [ ] Контекстные подсказки: обнаружение паттернов ("ты откладываешь 3-й раз")
- [ ] Оптимальный порядок выполнения задач
- [ ] Абстракция LLM-провайдера: переключение между OpenAI / Anthropic / другие
- [ ] Многоуровневая вложенность подзадач

### Phase 6 — Расширение (будущее)
> Цель: multi-user, интеграции, аналитика

- [ ] Канбан-доска внутри проектов (Mini App)
- [ ] Совместные списки / делегирование задач
- [ ] API/вебхуки для внешних интеграций
- [ ] Google Calendar синхронизация
- [ ] Аналитика продуктивности
- [ ] Мультиязычность (английский + автоопределение)

## 12. Стоимость AI-сервисов (оценка)

| Сервис | Стоимость | Использование |
|--------|-----------|---------------|
| Whisper API | ~$0.006/мин | Голосовые сообщения |
| GPT-4o-mini | ~$0.15/1M input, $0.60/1M output | Парсинг текста, категоризация |
| GPT-4o | ~$2.50/1M input, $10/1M output | Декомпозиция, сложные подсказки |
| Claude Haiku | ~$0.25/1M input, $1.25/1M output | Альтернатива для парсинга |
| Claude Sonnet | ~$3/1M input, $15/1M output | Альтернатива для сложных задач |

> При личном использовании (~50 задач/день) ожидаемые затраты: **< $1/месяц** на парсинг, **< $0.50/месяц** на голос.
