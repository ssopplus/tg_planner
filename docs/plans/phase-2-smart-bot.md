# Phase 2 — Умный бот

> Источник требований: [docs/DOMAIN.md](../DOMAIN.md)

## Цель

Бот становится умнее: понимает голос, формирует план дня, управляет просроченными задачами и отправляет контекстные уведомления с inline-кнопками. Пользователь получает утренний/вечерний дайджест и может управлять задачами прямо из уведомлений.

## Scope

- Голосовой ввод через Whisper API
- Утренний и вечерний дайджест
- "Мой день" — автоформирование и ручная корректировка
- Автоприоритет по дедлайнам
- Управление просроченными задачами
- Повторяющиеся напоминания
- Inline-кнопки на уведомлениях

## Пререквизиты

- Phase 1 полностью завершена
- Бот работает на Vercel с PostgreSQL
- AI-парсинг текста работает

## Задачи

### 1. Whisper API интеграция для голосовых сообщений

**Описание:** Бот принимает голосовые сообщения, транскрибирует через Whisper и создаёт задачи.

**Технические решения:**
- Telegram отправляет голосовые как OGG/Opus файлы
- Скачивание файла через Telegram Bot API `getFile`
- Отправка в OpenAI Whisper API (`whisper-1`)
- Результат транскрипции передаётся в существующий AI-парсер

**Структура:**
```
src/bot/handlers/
  voice.ts              # Обработчик голосовых сообщений
src/lib/
  speech/
    whisper.ts          # Whisper API клиент
    types.ts
```

**Поведение:**
1. Пользователь отправляет голосовое
2. Бот: "Распознаю речь..." (typing indicator)
3. Whisper транскрибирует
4. Текст передаётся в AI-парсер
5. Показ результата с подтверждением (как в Phase 1)

### 2. Умный парсинг нескольких задач из одного сообщения

**Описание:** Одно сообщение (текст или голос) может содержать несколько задач.

**Технические решения:**
- Обновление промпта AI-парсера: возвращает массив `ParsedTask[]`
- LLM определяет границы задач в тексте
- Подтверждение: показ всех распарсенных задач с общими кнопками [Всё верно] [Изменить] и индивидуальными кнопками для каждой задачи

**Пример:**
- Ввод: "Завтра надо купить молоко и позвонить маме в три часа"
- Результат: 2 задачи — "Купить молоко" (завтра) и "Позвонить маме" (завтра 15:00)

**Зависимости:** Задача 1 (голос), существующий AI-парсер из Phase 1

### 3. Утренний и вечерний дайджест

**Описание:** Автоматические сообщения утром (план дня) и вечером (итоги).

**Технические решения:**
- Vercel Cron Job: утренний — `0 8 * * *`, вечерний — `0 21 * * *` (UTC, пересчёт по timezone пользователя)
- Или: отдельный cron каждые 15 минут, проверяющий `users.digest_time` с учётом timezone
- Endpoint: `/api/cron/digest`

**Утренний дайджест включает:**
- Задачи на сегодня (с дедлайном сегодня)
- Просроченные задачи с кнопками [Перенести] [Удалить] [Выполнить]
- Задачи без даты с высоким приоритетом
- Общее количество задач

**Вечерний итог включает:**
- Сколько задач выполнено за день
- Что осталось / перенесено на завтра
- Краткая статистика

**Структура:**
```
src/bot/services/
  digest.ts             # Формирование дайджестов
  digest-morning.ts     # Утренний дайджест
  digest-evening.ts     # Вечерний итог
src/app/api/cron/
  digest/route.ts       # Cron endpoint
```

### 4. "Мой день" — автоформирование

**Описание:** Бот автоматически формирует список задач на сегодня.

**Технические решения:**
- Алгоритм формирования "Моего дня":
  1. Задачи с жёстким дедлайном сегодня (обязательные)
  2. Просроченные задачи (перенесённые)
  3. Задачи с мягким дедлайном, попадающим на сегодня
  4. Задачи с высоким приоритетом без даты
  5. Лимит: разумное количество (5-7 задач)
- Новое поле в tasks: `my_day_date` (DATE) — дата, на которую задача помещена в "Мой день"
- Команда `/today` — показывает "Мой день"
- Ручная корректировка: кнопки [Убрать из дня] / [Добавить в день]

**Зависимости:** Задача 3 (дайджест использует "Мой день")

### 5. Автоприоритет по дедлайнам

**Описание:** Автоматическое ранжирование задач на основе дедлайнов и контекста.

**Технические решения:**
- Алгоритм скоринга:
  - Жёсткий дедлайн сегодня: +100
  - Жёсткий дедлайн завтра: +70
  - Просрочено: +90
  - Мягкий дедлайн "эта неделя": +40
  - Ручной приоритет HIGH: +50, MEDIUM: +30, LOW: +10
  - Давно создана и не выполнена: +5 за каждый день
- Score хранится в вычисляемом поле (не в БД) — пересчитывается при запросе
- Сортировка в "Мой день" и списках по этому score

### 6. Просроченные задачи: автоперенос + утренний опрос

**Описание:** Задачи с истёкшим дедлайном обрабатываются автоматически.

**Технические решения:**
- Cron Job (утренний): находит задачи где `deadline_at < today` и `status = TODO`
- Автоматически добавляет в "Мой день"
- В утреннем дайджесте — отдельная секция с кнопками:
  - [Перенести на сегодня] — обновляет `deadline_at`
  - [Перенести на завтра]
  - [Выполнено]
  - [Удалить]
- Новое поле tasks: `overdue_count` (INT) — сколько раз переносилась

**Зависимости:** Задача 3 (дайджест), Задача 5 (автоприоритет)

### 7. Повторяющиеся напоминания (rrule)

**Описание:** Поддержка повторяющихся напоминаний.

**Технические решения:**
- Библиотека: `rrule` (RFC 5545 стандарт повторений)
- Новые поля в reminders: `rrule` (STRING, nullable), `is_recurring` (BOOL)
- При срабатывании повторяющегося напоминания: автоматически создаётся следующее по rrule
- Парсинг из естественного языка: "каждый понедельник в 9:00", "каждый день", "каждую пятницу"
- Обновление AI-промпта для распознавания повторений

**Структура:**
```
src/lib/
  reminders/
    scheduler.ts        # Логика планирования
    rrule-parser.ts     # Парсинг rrule из AI-результата
```

### 8. Inline-кнопки на уведомлениях

**Описание:** Каждое уведомление приходит с кнопками быстрых действий.

**Кнопки по типу уведомления:**
- Напоминание о задаче: [Готово] [Отложить на 1ч] [Отложить на завтра]
- Приближение дедлайна: [Готово] [Открыть задачу]
- Просроченная задача: [Перенести] [Выполнено] [Удалить]
- Дайджест: кнопки на каждой задаче в списке

**Технические решения:**
- Callback data формат: `action:entity_id:param`
- Обработчик callback queries с маршрутизацией по action
- Обновление сообщения после действия (edit message + обновлённый текст)

**Зависимости:** Задача 3 (дайджест), Задача 7 (повторяющиеся напоминания)

## Изменения в БД (миграции)

- tasks: добавить `my_day_date` (DATE, nullable), `overdue_count` (INT, default 0)
- reminders: добавить `rrule` (TEXT, nullable), `is_recurring` (BOOLEAN, default false)
- users: добавить `morning_digest_time` (TIME), `evening_digest_time` (TIME)

## Зависимости между задачами

```
1. Whisper API
   └── 2. Умный парсинг (расширение)

3. Дайджест
   ├── 4. "Мой день" (используется в дайджесте)
   ├── 5. Автоприоритет (сортировка в дайджесте)
   └── 6. Просроченные задачи (секция в дайджесте)

8. Inline-кнопки на уведомлениях (после 3, 7)
```

## Критерии готовности

- [ ] Голосовое сообщение распознаётся и создаёт задачу (включая несколько задач из одного сообщения)
- [ ] Утренний дайджест приходит в настроенное время с планом на день
- [ ] Вечерний итог показывает статистику дня
- [ ] `/today` показывает автоматически сформированный "Мой день"
- [ ] Задачи отсортированы по автоприоритету
- [ ] Просроченные задачи обнаруживаются и пользователю предлагается действие
- [ ] Повторяющиеся напоминания работают ("каждый понедельник в 9:00")
- [ ] Все уведомления приходят с inline-кнопками для быстрых действий
- [ ] Whisper API cost < $0.50/месяц при обычном использовании
