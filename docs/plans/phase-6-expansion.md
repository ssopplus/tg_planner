# Phase 6 — Расширение

> Источник требований: [docs/DOMAIN.md](../DOMAIN.md)

Расширение платформы: канбан-доска в Mini App, multi-user с совместными списками, внешние интеграции (API, Google Calendar), аналитика продуктивности и мультиязычность. Переход от личного инструмента к полноценной платформе.

## Scope

- Канбан-доска внутри проектов
- Multi-user: совместные списки и делегирование
- API/вебхуки для внешних интеграций
- Google Calendar синхронизация
- Аналитика продуктивности
- Мультиязычность (EN + автоопределение)

## Пререквизиты

- Phase 1-5 полностью завершены
- Стабильная архитектура, проверенная на одном пользователе
- Mini App и бот работают без критических багов

## Задачи

### 1. Канбан-доска в Mini App

**Описание:** Визуальное отображение задач проекта в виде канбан-колонок.

**Технические решения:**
- Колонки по статусам: TODO → IN_PROGRESS → DONE
- Библиотека: `@dnd-kit/core` (уже используется для "Мой день")
- Drag-n-drop между колонками — изменение статуса задачи
- Drag-n-drop внутри колонки — изменение порядка
- Опционально: пользовательские колонки (кастомные статусы)

**Поведение:**
- Переключатель: список ↔ канбан (на странице проекта)
- WIP-лимит: визуальное предупреждение если в IN_PROGRESS > N задач
- Мобильный UX: горизонтальный скролл между колонками
- Счётчик задач на каждой колонке

**Изменения в БД:**
- tasks: добавить `board_position` (INT, nullable) — позиция в колонке
- Опционально: новая таблица **board_columns** для кастомных колонок

**Структура:**
```
src/components/
  kanban/
    board.tsx             # Канбан-доска
    column.tsx            # Колонка
    card.tsx              # Карточка задачи
```

**Зависимости:** Нет (первая задача фазы)

### 2. Multi-user: совместные списки и делегирование

**Описание:** Несколько пользователей могут работать с общими проектами и задачами.

**Технические решения:**
- Новая таблица: **project_members** — project_id, user_id, role (OWNER, EDITOR, VIEWER), invited_at
- Новые поля в tasks: `assigned_to` (FK → users, nullable), `created_by` (FK → users)
- Инвайт: пользователь делится ссылкой или Telegram username
- Deep link: `t.me/bot?start=invite_{project_id}_{token}`

**Роли:**
- OWNER — полный доступ, может приглашать/удалять
- EDITOR — CRUD задач, но не настройки проекта
- VIEWER — только просмотр

**Делегирование задач:**
- Назначение задачи другому пользователю проекта
- Уведомление назначенному: "Тебе назначена задача X"
- Фильтр: "Мои задачи" / "Назначенные мне" / "Все"

**Совместные списки покупок:**
- Реалтайм: один отмечает "куплено", другой видит обновление
- Уведомление: "Мария купила: молоко, хлеб"

**Зависимости:** Нет (независимая крупная задача)

### 3. API и вебхуки для внешних интеграций

**Описание:** REST API и webhook-система для интеграции с внешними сервисами.

**Технические решения:**
- REST API с аутентификацией по API-ключу
- Endpoints: `/api/v1/tasks`, `/api/v1/projects`, `/api/v1/reminders`
- API-ключи: таблица **api_keys** — id, user_id, key (hashed), name, permissions, created_at, last_used_at
- Rate limiting: 100 req/min per key

**Вебхуки (исходящие):**
- Новая таблица: **webhooks** — id, user_id, url, events (JSON array), secret, is_active
- События: task.created, task.completed, task.updated, task.deleted, reminder.triggered
- Payload: JSON с данными события, подпись HMAC-SHA256
- Retry: 3 попытки с exponential backoff

**Документация:**
- OpenAPI/Swagger spec
- Страница в Mini App: управление API-ключами и вебхуками

**Зависимости:** Нет (независимая задача)

### 4. Google Calendar синхронизация

**Описание:** Двусторонняя синхронизация задач с дедлайнами и Google Calendar.

**Технические решения:**
- Google OAuth 2.0 для авторизации
- Google Calendar API v3
- Синхронизация:
  - TG Planer → Calendar: задачи с жёстким дедлайном создаются как события
  - Calendar → TG Planer: новые события создают задачи (опционально)
- Новая таблица: **calendar_sync** — id, user_id, google_account, calendar_id, sync_token, last_synced_at
- Новая таблица: **task_calendar_links** — task_id, google_event_id

**Поведение:**
- Синхронизация по триггеру (при создании/изменении задачи)
- Периодическая полная синхронизация (cron каждые 15 минут)
- Конфликты: последнее изменение выигрывает (last-write-wins)
- Настройка: какие проекты синхронизировать

**Зависимости:** Задача 3 (API-инфраструктура)

### 5. Аналитика продуктивности

**Описание:** Статистика и визуализация продуктивности пользователя.

**Технические решения:**
- Данные: из **user_activity_log** (Phase 5) + tasks history
- Метрики:
  - Задач выполнено за день/неделю/месяц
  - Среднее время выполнения задачи
  - Streak: дни подряд с выполненными задачами
  - Процент выполнения плана дня
  - Самые продуктивные дни недели / время суток
  - Распределение по проектам

**UI в Mini App:**
- Страница "Аналитика" с графиками
- Библиотека: `recharts` или `Chart.js` (лёгкая)
- Карточки с ключевыми метриками
- График активности (contribution graph, как GitHub)
- Streak-counter с визуальным эффектом

**В боте:**
- Команда `/stats` — краткая сводка
- В вечернем дайджесте: streak и ключевые метрики

**Структура:**
```
src/lib/analytics/
  metrics.ts              # Вычисление метрик
  streaks.ts              # Логика стриков
src/app/(mini-app)/
  analytics/page.tsx      # Страница аналитики
src/components/
  analytics/
    stats-cards.tsx
    activity-chart.tsx
    streak-counter.tsx
    project-breakdown.tsx
```

**Зависимости:** Нет (но использует данные из Phase 5)

### 6. Мультиязычность

**Описание:** Поддержка английского языка и автоопределение.

**Технические решения:**
- Библиотека i18n: `next-intl` (для Next.js) или `i18next`
- Языки: русский (default), английский
- Автоопределение: из `Telegram.WebApp.initDataUnsafe.user.language_code`
- Ручной выбор в настройках

**Что переводится:**
- Интерфейс Mini App (все строки)
- Сообщения бота (дайджесты, подтверждения, команды)
- Кнопки и клавиатуры
- AI-промпты (для корректного парсинга на английском)
- Категории покупок

**Структура:**
```
src/i18n/
  config.ts               # Конфигурация
  messages/
    ru.json               # Русские переводы
    en.json               # Английские переводы
  bot-messages/
    ru.ts                 # Сообщения бота (RU)
    en.ts                 # Сообщения бота (EN)
```

**Зависимости:** Нет (но затрагивает все предыдущие фазы)

