## Контекст

Пятая фаза развития TG Planer (см. [docs/DOMAIN.md](../../DOMAIN.md)). Бот становится умным помощником: декомпозирует сложные задачи, замечает паттерны поведения, предлагает оптимальный порядок выполнения. LLM-абстракция расширяется для нескольких провайдеров. Подзадачи получают многоуровневую вложенность.

### Вопросы и ответы

**Q:** Какую модель использовать для декомпозиции?
**A:** Мощную модель (GPT-4o / Claude Sonnet) — задача требует рассуждения, дешёвая модель не справится.

**Q:** Как определять паттерны поведения?
**A:** Набор rule-based правил на основе user_activity_log, без ML — проще и прозрачнее.

**Q:** Сколько уровней вложенности подзадач?
**A:** Максимум 3 уровня — больше ухудшает читаемость.

## Решение

AI-декомпозиция через мощную LLM-модель по запросу пользователя. Контекстные подсказки через rule-based анализатор паттернов (не ML). Оптимальный порядок выполнения — гибрид алгоритмического скоринга и LLM. Расширение LLM-абстракции: добавление Anthropic провайдера с фабрикой и конфигурацией задача→провайдер. Подзадачи с `parent_subtask_id` для вложенности до 3 уровней.

### Альтернативы

1. **ML для анализа паттернов** — overkill для одного пользователя, rule-based проще
2. **Только алгоритмическая оптимизация порядка** — не учитывает контекст (LLM лучше)
3. **Неограниченная вложенность подзадач** — ухудшает UX, сложнее рекурсивный расчёт

## Последствия

### Позитивные
- AI-декомпозиция решает проблему "большая задача, не знаю с чего начать"
- Контекстные подсказки помогают формировать продуктивные привычки
- Несколько LLM-провайдеров — устойчивость к сбоям и выбор оптимального
- Многоуровневые подзадачи для сложных проектов

### Риски
- AI-декомпозиция может генерировать нерелевантные подзадачи — нужно подтверждение
- Подсказки могут раздражать — лимит 1 в день обязателен
- Anthropic API может отличаться по формату ответов — нужна нормализация

### Технический долг
- Rule-based паттерны — жёсткие пороговые значения (вынести в конфиг)
- Нет A/B тестирования LLM провайдеров
- user_activity_log будет расти — нужна стратегия retention

## Тестирование

- "Разбей на подзадачи" для задачи "Сделать ремонт" → адекватные шаги
- Задача перенесена 3 раза → подсказка "Может, разбить на части?"
- Задача не трогалась 14 дней → подсказка "Актуально ли это?"
- "Оптимизировать порядок" → задачи переупорядочены с объяснением
- Anthropic провайдер работает для парсинга и декомпозиции
- Fallback: отключить OpenAI → автопереключение на Anthropic
- Подзадачи 3 уровней отображаются корректно в Mini App и боте
