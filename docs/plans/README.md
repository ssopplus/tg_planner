# Development Plans

Планы развития TG Planer. Источник требований: [docs/DOMAIN.md](../DOMAIN.md).

## Структура

```
docs/plans/
├── README.md          # Этот файл
├── STATUS.md          # Статус всех планов
├── active/            # Планы в работе
├── backlog/           # Планы на будущее
├── phase-*.md         # Детальные описания фаз (справочные)
```

## Workflow

1. **Planning** — план создан, задачи уточняются
2. **In Progress** — план выполняется
3. **Completed** — план завершён, перемещён в ADR (`docs/ADRs/`)

## Фазы развития

1. **Phase 1 — MVP** — бот с AI-парсингом и напоминаниями
2. **Phase 2 — Умный бот** — голос, дайджесты, планирование дня
3. **Phase 3 — Mini App** — визуальный интерфейс
4. **Phase 4 — Умный список покупок** — покупки как smart-проект
5. **Phase 5 — AI-фишки** — декомпозиция, подсказки, паттерны
6. **Phase 6 — Расширение** — multi-user, интеграции, аналитика
