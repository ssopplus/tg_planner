# Phase 3 — Mini App

> Источник требований: [docs/DOMAIN.md](../DOMAIN.md)

## Цель

Полноценный визуальный интерфейс через Telegram Mini App: просмотр и управление задачами, планирование дня с drag-n-drop, подзадачи с прогрессом и архив. Пользователь получает удобный UI в дополнение к чат-интерфейсу.

## Scope

- Telegram Mini App через WebApp API
- Страницы: задачи, проекты, "Мой день", настройки
- Drag-n-drop для планирования дня
- Подзадачи (один уровень) с прогрессом
- Архив выполненных задач

## Пререквизиты

- Phase 1 и Phase 2 завершены
- API endpoints для CRUD уже работают (расширяем)
- Пользователи, проекты, задачи, "Мой день" — всё в БД

## Задачи

### 1. Telegram Mini App setup

**Описание:** Настройка Mini App инфраструктуры и авторизации.

**Технические решения:**
- Telegram Mini App открывается через кнопку в боте или Menu Button
- Авторизация: `WebApp.initData` → серверная валидация HMAC
- Next.js: Mini App страницы в `src/app/(mini-app)/`
- Передача `initData` через HTTP заголовок при API-вызовах
- Middleware для проверки авторизации на API routes

**Структура:**
```
src/app/(mini-app)/
  layout.tsx              # Mini App layout + Telegram WebApp SDK init
  page.tsx                # Главная (редирект на /today)
  today/page.tsx          # "Мой день"
  tasks/page.tsx          # Все задачи
  tasks/[id]/page.tsx     # Детали задачи
  projects/page.tsx       # Проекты
  projects/[id]/page.tsx  # Задачи проекта
  settings/page.tsx       # Настройки
  archive/page.tsx        # Архив
src/lib/
  telegram/
    webapp.ts             # WebApp SDK утилиты
    auth.ts               # Валидация initData
src/app/api/
  tasks/route.ts          # REST API для задач
  projects/route.ts       # REST API для проектов
  today/route.ts          # API для "Мой день"
```

**Зависимости:** Нет (первая задача фазы)

### 2. UI-стек: shadcn/ui + Tailwind

**Описание:** Настройка дизайн-системы для Mini App.

**Технические решения:**
- shadcn/ui — компонентная библиотека (не зависимость, копируется в проект)
- Tailwind CSS — утилитарные стили
- Адаптация темы под Telegram: использование `WebApp.themeParams` для цветов
- CSS переменные: маппинг Telegram theme → shadcn theme tokens
- Компоненты: Button, Card, Dialog, DropdownMenu, Input, Badge, Checkbox, Sheet (bottom sheet)

**Структура:**
```
src/components/
  ui/                     # shadcn/ui компоненты
  layout/
    nav-bar.tsx           # Нижняя навигация
    header.tsx            # Верхний заголовок с back button
  tasks/
    task-card.tsx         # Карточка задачи
    task-list.tsx         # Список задач
    task-detail.tsx       # Детальный просмотр
    task-form.tsx         # Форма создания/редактирования
  projects/
    project-card.tsx
    project-list.tsx
```

**Зависимости:** Задача 1 (Mini App setup)

### 3. Страница "Мой день" с drag-n-drop

**Описание:** Главная страница Mini App — план на сегодня с возможностью переупорядочивания.

**Технические решения:**
- Библиотека drag-n-drop: `@dnd-kit/core` + `@dnd-kit/sortable` (лёгкая, touch-friendly)
- Список задач "Моего дня" с сортировкой перетаскиванием
- Сохранение порядка: поле `sort_order` в tasks (для my_day)
- Кнопка "Добавить задачу в день" — поиск/выбор из всех задач
- Свайп-действия: свайп влево — завершить, свайп вправо — убрать из дня
- Pull-to-refresh

**API:**
- `GET /api/today` — задачи "Моего дня" отсортированные
- `PATCH /api/today/reorder` — обновление порядка
- `POST /api/today/add` — добавить задачу в "Мой день"
- `DELETE /api/today/:id` — убрать из "Моего дня"

**Зависимости:** Задача 2 (UI-стек)

### 4. Страница задач и проектов

**Описание:** Просмотр всех задач с фильтрацией по проектам.

**Функциональность:**
- Список задач с группировкой по проектам
- Фильтры: все / по проекту / по статусу / по приоритету
- Сортировка: по дате создания / дедлайну / приоритету
- Быстрые действия: checkbox для завершения, swipe для удаления
- Создание новой задачи: bottom sheet с формой
- FAB (Floating Action Button) для быстрого создания

**Страница проектов:**
- Список проектов с количеством задач
- Создание/редактирование/удаление проекта
- Переход к задачам проекта

**API:**
- `GET /api/tasks?project_id=&status=&sort=`
- `POST /api/tasks`
- `PATCH /api/tasks/:id`
- `DELETE /api/tasks/:id`
- `GET /api/projects`
- `POST /api/projects`
- `PATCH /api/projects/:id`
- `DELETE /api/projects/:id`

**Зависимости:** Задача 2 (UI-стек)

### 5. Страница деталей задачи и редактирование

**Описание:** Полный просмотр и редактирование задачи.

**Функциональность:**
- Название (inline-edit)
- Описание (textarea)
- Проект (select)
- Приоритет (select/badges)
- Дедлайн: дата + время picker
- Тип дедлайна: жёсткий / мягкий
- Напоминания: список с добавлением
- Подзадачи: чеклист с прогрессом (см. задачу 6)
- Кнопки: [Завершить] [В архив] [Удалить]
- Telegram BackButton для навигации назад

**Зависимости:** Задача 4 (список задач)

### 6. Подзадачи с прогрессом

**Описание:** Один уровень подзадач с отображением прогресса на карточке задачи.

**Технические решения:**
- Новая таблица: **subtasks** — id, task_id (FK), title, is_completed, sort_order, created_at
- Прогресс вычисляется: "выполнено 3 из 7"
- На карточке задачи: progress bar + текст
- В деталях задачи: список подзадач с чекбоксами
- Добавление подзадач: inline-input внизу списка
- Drag-n-drop для переупорядочивания подзадач

**API:**
- `GET /api/tasks/:id/subtasks`
- `POST /api/tasks/:id/subtasks`
- `PATCH /api/subtasks/:id`
- `DELETE /api/subtasks/:id`
- `PATCH /api/tasks/:id/subtasks/reorder`

**Зависимости:** Задача 5 (детали задачи)

### 7. Настройки

**Описание:** Страница пользовательских настроек.

**Параметры:**
- Часовой пояс (auto-detect + ручной выбор)
- Время утреннего дайджеста
- Время вечернего итога
- Проект по умолчанию
- Включение/выключение уведомлений

**Зависимости:** Задача 2 (UI-стек)

### 8. Архив выполненных задач

**Описание:** Просмотр истории выполненных задач.

**Функциональность:**
- Список задач со статусом ARCHIVED / DONE
- Фильтрация по проекту и дате
- Возможность восстановить задачу (вернуть в TODO)
- Infinite scroll / pagination
- Поиск по названию

**Технические решения:**
- При завершении задачи: status → DONE, completed_at = now()
- Автоархивация: задачи DONE старше 7 дней → ARCHIVED (cron)
- API: `GET /api/tasks?status=DONE,ARCHIVED&page=&limit=`

**Зависимости:** Задача 4 (список задач)
