# Phase 4 — Умный список покупок

> Источник требований: [docs/DOMAIN.md](../DOMAIN.md)

## Цель

Покупки как отдельный smart-проект с быстрым вводом, автокатегоризацией по отделам магазина и запоминанием частых покупок. Пользователь вводит "молоко, хлеб, яйца" — получает структурированный список, сгруппированный по категориям.

## Scope

- Тип проекта "Покупки" с расширенным поведением
- Быстрый ввод списком через запятую
- Автокатегоризация товаров по отделам
- Запоминание частых покупок с предложениями

## Пререквизиты

- Phase 1-3 завершены
- CRUD задач и проектов работают в боте и Mini App
- AI-парсинг настроен

---

## Задачи

### 1. Тип проекта "Покупки"

**Описание:** Проекты с типом SHOPPING получают расширенное поведение и отдельный UI.

**Технические решения:**
- Поле `type` в projects уже есть (enum: DEFAULT, SHOPPING) — из Phase 1
- При создании проекта с типом SHOPPING — другой набор полей и UI
- В боте: команда `/shopping` или распознавание контекста ("купить молоко" → проект Покупки)
- В Mini App: отдельный вид для проекта SHOPPING с категориями

**Поведение задач в shopping-проекте:**
- Нет дедлайнов (покупки не имеют срока)
- Нет приоритетов (порядок определяется категорией)
- Завершение = "куплено" (визуальное отличие)
- Завершённые товары скрываются, но остаются в истории

**Зависимости:** Нет (первая задача фазы)

### 2. Быстрый ввод списком

**Описание:** Ввод нескольких товаров одной строкой через запятую или перенос строки.

**Технические решения:**
- Парсинг на клиенте и сервере: разделение по `,`, `\n`, ` и `
- Количество: "молоко 2л", "яйца 10шт" — парсинг количества/единиц
- AI-парсинг: для неоднозначных случаев, "что-нибудь к чаю" → печенье, сахар, лимон (опционально)

**В боте:**
- Сообщение в SHOPPING-проект: "молоко, хлеб, яйца 10шт, сметана" → 4 позиции
- Подтверждение: показ списка + [Добавить] [Изменить]

**В Mini App:**
- Textarea для быстрого ввода
- Preview списка перед добавлением
- Кнопка "Добавить всё"

**API:**
- `POST /api/shopping/:projectId/items` — принимает массив или строку для парсинга
- Ответ: массив созданных элементов с категориями

**Зависимости:** Задача 1 (тип проекта)

### 3. Автокатегоризация товаров

**Описание:** Автоматическая группировка товаров по отделам магазина.

**Технические решения:**
- **Гибридный подход:**
  1. Словарь — mapping частых товаров по категориям (быстро, бесплатно)
  2. LLM fallback — для товаров не найденных в словаре

- **Категории (предустановленные):**
  - Молочные продукты
  - Хлеб и выпечка
  - Мясо и рыба
  - Овощи и фрукты
  - Бакалея (крупы, макароны)
  - Напитки
  - Замороженные продукты
  - Бытовая химия
  - Личная гигиена
  - Другое

- Новая таблица: **shopping_categories** — id, name, sort_order, icon
- Новые поля в tasks (для SHOPPING проектов): `category_id` (FK, nullable)

**Словарь:**
```
src/lib/shopping/
  categories.ts           # Определения категорий
  dictionary.ts           # Mapping товар → категория
  categorizer.ts          # Логика: словарь → LLM fallback
```

**В Mini App:**
- Товары сгруппированы по категориям (аккордеон или секции)
- Пользователь может изменить категорию товара (система запоминает)

**Зависимости:** Задача 2 (быстрый ввод)

### 4. Запоминание частых покупок и предложения

**Описание:** Система запоминает что пользователь покупает и предлагает товары.

**Технические решения:**
- Новая таблица: **frequent_items** — id, user_id, name, category_id, purchase_count, last_purchased_at
- При завершении товара: `purchase_count++`, `last_purchased_at = now()`
- Нормализация названий: "Молоко" = "молоко" = "МОЛОКО" (lowercase + trim)

**Предложения:**
- В боте: после создания shopping-проекта — "Добавить частые покупки?" + список
- В Mini App: секция "Часто покупаете" с кнопками быстрого добавления
- Сортировка: по purchase_count DESC, потом по last_purchased_at DESC
- Показываем топ-10 частых покупок

**UI в Mini App:**
- Chips с частыми товарами над полем ввода
- Tap на chip = добавить в текущий список
- Уже добавленные чипы помечаются (disabled или иконкой)

**Зависимости:** Задача 3 (категоризация)

## Изменения в БД (миграции)

- Новая таблица: **shopping_categories** (id, name, sort_order, icon)
- Новая таблица: **frequent_items** (id, user_id, name, normalized_name, category_id, purchase_count, last_purchased_at)
- tasks: добавить `category_id` (FK → shopping_categories, nullable)
- Seed data: предустановленные категории товаров

```

src/components/
  shopping/
```

- [ ] Проект с типом "Покупки" создаётся и имеет отдельный UI
- [ ] Строка "молоко, хлеб, яйца" парсится в 3 отдельных товара
- [ ] Товары автоматически распределяются по категориям
- [ ] В Mini App товары сгруппированы по отделам (категориям)
- [ ] Частые покупки запоминаются и предлагаются при создании нового списка
- [ ] Пользователь может изменить категорию товара
- [ ] Словарь покрывает 80%+ типичных продуктовых товаров
- [ ] LLM fallback работает для неизвестных товаров
- [ ] В боте: "купить молоко" автоматически попадает в проект "Покупки"
